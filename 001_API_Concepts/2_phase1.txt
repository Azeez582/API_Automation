PHASE 1 — API FUNDAMENTALS
Lesson 1 — What is an API
Simple Explanation
API = Application Programming Interface

An API allows two software systems to communicate with each other.

Think of it like a messenger between systems.

Example:

Mobile App  →  API  →  Database
or

Frontend UI  →  API  →  Backend service
The API sends a request and receives a response.

Real Life Example
Imagine ordering food.

Customer → Waiter → Kitchen
Customer = Client
Waiter = API
Kitchen = Server / Database

Steps:

1️⃣ You order food
2️⃣ Waiter takes the order to the kitchen
3️⃣ Kitchen prepares food
4️⃣ Waiter returns food

Exactly how API works.

Technical Flow
Client → API Request → Server → Database
                       ↓
                API Response
Example:

Client sends:

GET /users/1
Server responds:

{
 "id":1,
 "name":"John",
 "email":"john@email.com"
}

Login System

API validates user and returns:

Response:

1️⃣ The URL Does NOT Contain the HTTP Method
This URL:

https://jsonplaceholder.typicode.com/comments/10
is only the endpoint.

It does not show the HTTP method.

The HTTP method is part of the request sent to the server, not the URL itself.

2️⃣ What Happens When You Open It in the Browser?
When you paste a URL in a browser:

https://jsonplaceholder.typicode.com/comments/10
The browser automatically sends a GET request.

Internally the browser sends:

GET /comments/10 HTTP/1.1
Host: jsonplaceholder.typicode.com
So the browser decides to use GET because it is trying to fetch data.

3️⃣ How QA Engineers Explicitly Send the GET Request
In automation we explicitly define the HTTP method.

Example using Python:

import requests

response = requests.get("https://jsonplaceholder.typicode.com/comments/10")

print(response.status_code)
print(response.json())
Here you clearly see:

requests.get()
This is where GET is defined.

Important Interview Question
How does the server know what action to perform?
Good answer:

The server determines the action based on the HTTP method (GET, POST, PUT, DELETE) and the endpoint provided in the request.

Example:

GET    /users     → retrieve users
POST   /users     → create user
PUT    /users/1   → update user
DELETE /users/1   → delete user
Real Industry Insight
Modern APIs often use the same endpoint but different HTTP methods.

Example:

GET    /users
POST   /users
Same endpoint:

/users
Different actions because of HTTP method.
